# YOPI LAND (요피 랜드) - 프로젝트 정의서

## 1. 프로젝트 개요
- **목표**: 최신 웹 기술을 실험하고 다양한 마이크로 서비스를 통합 제공하는 개인 개발 포털.
- **핵심 가치**: 높은 정보 밀도, 즉각적인 반응성(Optimistic UI), 철저한 예외 처리와 안전한 보안 체계.

## 2. 서비스 아키텍처
- **Dashboard-First**: 메인 화면에서 최신/인기 콘텐츠를 한눈에 파악하는 3단 대시보드 제공.
- **Modal-Based Interaction**: 상세 조회 시 페이지 이동 없이 모달을 통해 데이터를 즉시 로드.
- **Feed-Style Board**: 게시판 목록에서 본문과 댓글 섹션이 즉시 노출되는 피드 형태의 무한 소통 환경 제공.
- **Connection Efficiency**: Next.js HMR 환경에서의 DB 커넥션 누수를 방지하는 싱글톤 패턴 적용.

## 3. 기술 스택
- **Framework**: Next.js 15 (App Router, React 19)
- **Database**: Supabase (PostgreSQL) + Drizzle ORM
- **Storage**: Supabase Storage (Profile Avatars)
- **Auth**: Supabase Auth (Google OAuth)
- **Styling**: Tailwind CSS (Compact/Minimize UI)
- **Logic**: Server Actions + useOptimistic + Context API (Dialog System)
- **Optimization**: Native Intersection Observer (Infinite Scroll), Client-side Canvas Resizing

## 4. 핵심 기능 명세

### 4.1. 통합 인증 및 프로필
- **익명성 보장**: 첫 로그인 시 임의의 닉네임 부여 및 이메일 비노출.
- **프로필 관리**: 
  - **아바타(프로필 사진)**: 
    - 사진 업로드 및 변경 기능 지원.
    - 서버 저장 용량 및 전송 대역폭 최적화를 위해 클라이언트 측에서 `Canvas`를 이용해 **300x300 크기의 WebP** 형식으로 자동 리사이징 후 업로드.
  - **자기소개**: 닉네임(최대 10자) 및 한줄 자기소개(최대 100자) 설정.
  - **입력 제한**: 입력 시 실시간 글자 수 카운터 및 초과 방지(`maxLength`) 적용.
- **안전한 탈퇴**: 
  - Edge Functions를 통한 계정 정보 및 소셜 연동 해제.
  - **데이터 정책**: 탈퇴 시 개인정보 및 프로필은 즉시 삭제되나, 작성한 게시글/댓글은 사이트 운영을 위해 유지됨(작성자 정보는 익명화 처리).

### 4.2. 대시보드 (Dashboard - `/`)
- **3대 지표 집계**: 최근 30일 데이터를 기준으로 섹션 구성.
  - **최근 소식**: 전체 최신 게시글 5개.
  - **인기 게시글**: 좋아요 합계 상위 5개.
  - **활발한 논의**: 댓글 합계 상위 5개.
- **퀵 프리뷰**: 리스트 클릭 시 모달 팝업으로 상세 내용 및 댓글 즉시 로드.

### 4.3. 자유게시판 (Free Board - `/board`)
- **무한 스크롤 (Infinite Scroll)**:
  - 게시글 목록 조회 시 대량의 데이터를 한 번에 로드하지 않고 10개 단위로 페이징 조회.
  - Native `Intersection Observer`를 사용하여 외부 라이브러리 없이 가볍고 부드러운 스크롤 경험 제공.
- **플로팅 액션 버튼 (FAB)**:
  - 화면 우측 하단에 고정된 시그니처 컬러(#e2ff00) 버튼 제공.
  - '작성' 직관성을 높인 연필 아이콘과 호버 툴팁 적용.
- **구조화된 레이아웃 (개편)**:
  - **Identity Bar**: 작성자 정보(아바타+닉네임)와 관리 기능(수정/삭제)을 독립적인 바 형태로 상단에 배치하여 본문과 시각적으로 격리.
  - **정보 위계 최적화**: [제목] -> [작성 시간] -> [본문] 순으로 시선이 자연스럽게 흐르도록 배치하여 가독성 향상.
- **인라인 편집 (Inline CRUD)**: 
  - 게시글 수정 시 페이지 이동 없이 피드에서 즉시 입력 폼으로 전환.
  - `useOptimistic`을 통한 즉각적인 UI 반영 및 서버 액션 처리.
- **본문 가독성 관리**: 
  - 제목: 최대 50자 제한, 목록에서 2줄 초과 시 생략 처리.
  - 본문: 최대 5,000자 제한, 목록에서 5줄 초과 시 '더보기/접기' 기능 제공.
- **답글(댓글) 시스템**: 
  - **한 줄 소통**: 간결한 소통을 위해 줄 바꿈이 금지된 한 줄 입력(Input) 방식 채택.
  - **길이 제한**: 최대 200자 제한 및 실시간 카운터 제공.
- **양방향 좋아요**: 게시글 및 개별 댓글에 대한 하트 토글 기능.

### 4.4. 공통 UX/UI 가이드라인
- **컴포넌트 시스템**:
  - **UserAvatar**: 시스템 전반의 공통 아바타 컴포넌트. 4단계(sm, md, lg, xl) 크기 지원 및 그림자 효과 적용.
  - **UserNickname**: 독립적인 텍스트 링크 형태의 닉네임 컴포넌트.
- **커스텀 다이얼로그**: 브라우저 기본 `alert/confirm`을 배제하고, 디자인 테마와 일치된 전역 모달 시스템(`DialogProvider`) 사용.
- **방어적 UI**: 모든 입력창에 `maxLength`를 강제하여 한도 초과 입력을 원천 차단하고, 초과 시 제출 버튼 비활성화.

## 5. 로드맵
- **1단계 (기반 구축 - 완료)**: 통합 인증 및 프로필 시스템 확립.
- **2단계 (커뮤니티 고도화 - 완료)**: 피드형 게시판, 인라인 편집, 대시보드 집계, 모달 프리뷰, 전역 다이얼로그 시스템 구축.
- **3단계 (인프라 및 UX 확장 - 완료)**: 무한 스크롤 도입, 프로필 사진 시스템(리사이징 포함) 구축, 디자인 시스템 고도화(FAB, Identity Bar).
- **4단계 (서비스 다각화)**: 실시간 멀티플레이 실험 및 타 서비스 통합.
