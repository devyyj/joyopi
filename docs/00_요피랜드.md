# YOPI LAND (요피 랜드) - 프로젝트 정의서

## 1. 프로젝트 개요
- **도메인**: joyopi.com
- **목표**: 개발자의 개인 실험실이자 다양한 서비스를 통합 제공하는 놀이터(Portal).
- **핵심 가치**: 확장성, 통합 인증, 사용자 경험(UX) 실험.

## 2. 서비스 아키텍처
- **Portal (Gateway)**: 메인 화면에서 모든 서비스(게시판, 게임, 예약 등)로 진입 가능한 카드형 대시보드.
- **SSO (Single Sign-On)**: 구글 로그인을 통해 한 번의 로그인으로 모든 내부 서비스를 이용.
- **Service Isolation**: 각 서비스는 `/board`, `/games`, `/booking` 등 독립된 경로를 가지며 공통 레이아웃을 공유.

## 3. 기술 스택
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Auth**: Supabase Auth (Google OAuth)
- **Database**: Supabase (PostgreSQL) + Drizzle ORM
- **Styling**: Tailwind CSS
- **Testing**: Vitest + UI Testing

## 4. 상세 기능 명세

### 4.1. 통합 인증 및 계정 (Auth & Profile)
- **구글 로그인**: OAuth 2.0을 통한 간편 로그인 및 세션 관리.
- **자동 프로필 생성**: 첫 로그인 시 '형용사 + 명사 + 숫자' 조합의 임의 닉네임을 자동으로 부여 (사용자 개인정보 보호 및 즉시 활동 지원).
- **프로필 설정 (`/profile`)**:
  - **닉네임 수정**: 고유한 닉네임으로 변경 가능 (중복 검사 포함).
  - **자기소개**: 자신을 표현하는 간단한 텍스트 저장.
- **회원 탈퇴**: 사용자의 명시적 요청 시 프로필 및 관련 데이터를 삭제하고 세션 종료.

### 4.2. 자유게시판 (Free Board - `/board`)
- **목록 조회**: 전체 게시글을 최신순으로 열람.
- **상세 보기**: 게시글의 제목, 작성자(닉네임), 작성일, 본문 확인.
- **게시글 관리**:
  - **작성**: 로그인한 사용자만 가능하며, 프로필의 현재 닉네임이 작성자로 기록됨.
  - **수정/삭제**: 본인이 작성한 게시글에 한해 권한 부여 (서버 사이드 검증).
- **데이터 무결성**: 계정 탈퇴 시 해당 사용자가 작성한 모든 게시글이 연쇄 삭제(Cascade Delete)됨.

### 4.3. 공통 UI/UX
- **메인 헤더**: 
  - 로그인 전: 로그인 버튼 표시.
  - 로그인 후: 사용자의 닉네임 및 아바타(이니셜) 표시, 클릭 시 프로필 설정 페이지 이동.
- **반응형 디자인**: 모바일 및 데스크탑 환경 최적화.

## 5. 로드맵 및 향후 계획
- **1단계 (완료)**: 자유게시판 모듈 구축 및 통합 인증 체계 확립.
- **2단계**: 게시판 내 댓글 기능 및 이미지 업로드(Supabase Storage) 추가.
- **3단계**: 새로운 마이크로 서비스(예: 게임 또는 예약 시스템) 통합 실험.
