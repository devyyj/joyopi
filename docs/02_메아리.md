# 메아리 (Echo) - 상세 기능 명세 (v1.1)

## 1. 개요
'메아리'는 사용자 간의 실시간 역할 기반 상호작용을 지원하는 서비스입니다. 한 명 이상의 '스피커'와 다수의 '샌더'가 연결되어, 샌더의 요청에 따라 스피커의 브라우저에서 실시간으로 사운드가 재생되는 시스템입니다.

## 2. 핵심 기능

### 2.1. 역할 정의 (Roles)
- **스피커 (Speaker)**: 
  - 사운드를 실제로 재생하는 주체입니다. 
  - 샌더로부터 받은 요청 시간을 누적하여 사운드를 송출합니다.
  - **[업데이트]** 스피커 자신도 '메아리 보내기' 버튼을 통해 자신의 재생 시간을 연장할 수 있습니다.
- **샌더 (Sender)**: 
  - 스피커에게 사운드 재생 요청을 보내는 주체입니다. 
  - 버튼을 클릭할 때마다 모든 활성 스피커의 재생 시간을 연장시킵니다.

### 2.2. 실시간 사운드 및 타이머 제어
- **재생 자원**: `/public/footsteps.mp3` (발소리 효과음).
- **시간 누적 로직**:
  - '메아리 보내기' 클릭 시 마다 **1분(60초)**의 재생 시간이 추가됩니다.
  - 여러 명의 참여자가 동시에 클릭할 경우, 모든 요청 시간이 **합산**됩니다.
- **상태 동기화**: 
  - **스피커 -> 샌더**: 스피커는 초 단위로 현재 남은 시간을 브로드캐스트(`ECHO_SYNC`)합니다.
  - **샌더**: 낙관적 업데이트를 적용하여 클릭 즉시 로컬 타이머를 연장하고, 이후 스피커의 `ECHO_SYNC` 신호를 통해 서버 상태와 정합성을 맞춥니다.

### 2.3. 즉시 중지 (Emergency Stop)
- **권한**: 모든 참여자(스피커, 샌더)가 '즉시 중지' 버튼을 사용할 수 있습니다.
- **효과**: 클릭 즉시 모든 스피커의 오디오 재생이 중단되며, 모든 참여자의 화면에서 남은 시간이 0으로 초기화됩니다.

### 2.4. 이벤트 로그 (Log Viewer) - [신규]
- **목적**: 실시간 이벤트 흐름 파악 및 디버깅.
- **출력 위치**: 스피커 및 샌더 페이지 하단.
- **로그 내용**:
  - 접속/퇴장: `[ID] 사용자가 입장/퇴장했습니다. (이유: ...)`
  - 메아리 요청: `[ID] 사용자가 메아리를 보냈습니다. (+1분)`
  - 메아리 응답: `[스피커ID] 메아리를 재생합니다. (남은 시간: ...)`
  - 중지 이벤트: `[ID] 사용자가 모든 메아리를 중지시켰습니다.`

## 3. 기술적 구현

### 3.1. Supabase Realtime
- **Presence**: 참여자의 역할(`role`), 닉네임 또는 임시 ID, 접속 시간을 추적합니다.
- **Broadcast**:
  - `ECHO_REQUEST`: 재생 시간 추가 요청.
  - `ECHO_STOP`: 모든 재생 중단 신호.
  - `ECHO_SYNC`: 스피커의 현재 상태(남은 시간) 동기화 신호.

### 3.2. 사운드 제어
- **Audio 객체**: 브라우저 정책에 따라 사용자 상호작용(역할 선택, 버튼 클릭 등) 이후에 `play()`를 호출합니다.
- **루프 재생**: `loop = true` 설정을 통해 시간이 남아있는 동안 계속해서 재생합니다.

### 3.3. UX 가이드
- **반응형 디자인**: 모바일과 데스크톱 모두에서 큰 타이머와 버튼을 제공합니다.
- **접속 상태 표시**: 현재 대기 중인 스피커 수와 전체 참여자 수를 실시간으로 표시합니다.

## 4. 로드맵 및 정책
- **오디오 권한 안내**: 브라우저 차단 시 안내 메시지 노출.
- **성능 최적화**: 다수의 샌더 요청 시 브로드캐스트 메시지 스로틀링 고려.

## 5. 트러블슈팅 히스토리

### 5.1. [v1.2] 소켓 연결 지속성 관리 (2026-05-17)
- **증상**: 
  - '연결이 중단되었습니다' 경고 반복 노출 및 타이머 멈춤.
- **원인 분석**:
  1. **리소스 제한**: 백그라운드 탭에서 브라우저가 타이머 및 소켓 제한.
  2. **리스너 누수**: 재연결 시 이전 채널이 완전히 제거되지 않음.
  3. **서버 타임아웃**: 유휴 시간 동안 패킷 교환 부재로 인한 강제 종료.
- **조치 사항**:
  - `supabase.removeChannel()` 명시적 호출 추가.
  - Visibility API 및 Online 상태 감지를 통한 능동적 복구 트리거.
  - 지능형 백오프 재시도(Exponential Backoff) 및 상세 로깅 강화.

### 5.2. [v1.3] 소켓 중복 연결 방지 및 구조 고도화 (2026-05-18)
- **증상**: 
  - 네트워크 불안정 시 중복 재연결 시도로 인한 시스템 과부하.
  - 스피커/샌더 로직 파편화로 인한 유지보수 어려움.
- **원인 분석**:
  - `CHANNEL_ERROR`와 `CLOSED` 동시 발생 시 병렬 재연결 로직 가동.
  - 리액트 렌더링 사이클과 소켓 구독 시점의 불일치로 인한 리스너 중첩.
- **조치 사항**:
  - **`useEchoSocket` 커스텀 훅 도입**: 소켓 관리 로직을 일원화하여 파편화 해결.
  - **Ref 기반 단일 연결 보장**: `reconnectTimeoutRef`를 도입하여 새로운 연결 시도 전 기존 시도를 반드시 취소.
  - **구독 최적화**: `onBroadcastRef`를 사용하여 콜백 변경 시에도 채널 재설정 없이 최신 로직을 참조하도록 개선.
  - **상태 동기화 강화**: 하트비트 주기를 최적화하고 가시성 복구 시점에 채널 상태를 선제적으로 점검.
