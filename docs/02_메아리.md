# 메아리 (Echo) - 상세 기능 명세

## 1. 개요
'메아리'는 사용자 간의 실시간 역할 기반 상호작용을 지원하는 서비스입니다. 한 명의 '스피커'와 다수의 '샌더'가 연결되어, 샌더의 요청에 따라 스피커의 브라우저에서 실시간으로 사운드가 재생되는 시스템입니다.

## 2. 핵심 기능

### 2.1. 역할 선택 (Role Selection)
- **스피커 (Speaker)**: 사운드를 실제로 재생하는 주체입니다. 샌더로부터 받은 요청 시간을 누적하여 사운드를 송출합니다.
- **샌더 (Sender)**: 스피커에게 사운드 재생 요청을 보내는 주체입니다. 버튼을 클릭할 때마다 스피커의 재생 시간을 연장시킵니다.

### 2.2. 실시간 사운드 제어
- **재생 자원**: `/public/footsteps.mp3` (발소리 효과음).
- **시간 누적 로직**:
  - 샌더가 '메아리 보내기' 버튼을 클릭할 때마다 **1분(60초)**의 재생 시간이 추가됩니다.
  - 여러 명의 샌더가 동시에 클릭할 경우, 모든 요청 시간이 **합산**됩니다. (예: 3명이 클릭 시 3분 재생)
- **상태 동기화**: 
  - 현재 남은 재생 시간이 스피커와 샌더 모두에게 실시간으로 초 단위로 표시됩니다.
  - **낙관적 업데이트(Optimistic UI)**: 샌더가 버튼을 누르는 즉시 로컬 타이머를 1분 연장하여 사용자에게 즉각적인 피드백을 제공합니다. 이후 스피커의 동기화 신호(`ECHO_SYNC`)에 의해 실제 서버 상태와 정합성을 맞춥니다.

### 2.3. 즉시 중지 (Emergency Stop)
- **권한**: 스피커와 샌더 모두 '즉시 중지' 버튼을 사용하여 현재 진행 중인 모든 재생 요청을 취소하고 잔여 시간을 0으로 초기화할 수 있습니다.
- **효과**: 클릭 즉시 스피커의 오디오 재생이 중단되며, 모든 참여자의 화면에서 남은 시간이 초기화됩니다.

## 3. 기술적 구현

### 3.1. Supabase Realtime (Presence & Broadcast)
- **상태 관리 (Presence)**: 현재 스피커로 활동 중인 사용자가 있는지 확인하고, 참여자들의 역할을 추적합니다.
- **신호 전달 (Broadcast)**:
  - `ECHO_REQUEST`: 샌더가 전송. 재생 시간 60초 추가 신호.
  - `ECHO_STOP`: 스피커/샌더가 전송. 모든 재생 즉시 중단 신호.
  - `ECHO_SYNC`: 스피커가 전송. 현재 남은 시간을 모든 샌더에게 브로드캐스트하여 UI 동기화.

### 3.2. 사운드 루프 재생
- **HTML5 Audio / Web Audio API**: 1분간의 사운드를 루프(Loop) 형태로 재생하며, 남은 시간이 0이 되면 자동으로 정지합니다.
- **에셋 위치**: `/footsteps.mp3`

### 3.3. UI/UX 디자인 가이드
- **내비게이션**: 상단 헤더에 '에코' 메뉴를 추가하여 접근성을 보장합니다.
- **타이머 인터페이스**: 남은 시간을 직관적으로 보여주는 대형 디지털 타이머를 중앙에 배치합니다.
- **역할별 뷰**: 
  - 스피커: 재생 중인 상태와 볼륨 제어, 중지 버튼 노출.
  - 샌더: 메아리 전송 버튼, 잔여 시간 확인, 중지 버튼 노출.

## 4. 로드맵 및 정책
- **오디오 권한**: 브라우저 보안 정책상 사용자의 첫 클릭(역할 선택 등) 이후에만 오디오 재생이 가능함을 안내합니다.
- **안티 스팸**: 단시간 내 과도한 요청을 방지하기 위해 샌더별 송신 쿨타임을 적용할 수 있습니다.
