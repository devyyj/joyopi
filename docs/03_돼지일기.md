# 돼지 일기 (Piggy Diary) - 상세 기능 명세 (v1.0 - Draft)

## 1. 개요
'돼지 일기'는 사용자가 매일 먹은 음식과 식사를 기록하고, 이를 기반으로 다채롭고 직관적인 식생활 통계를 제공하는 조요피 연구소의 웰니스/라이프 스타일 실험 서비스입니다. 단순히 칼로리 기록에만 집중하는 딱딱한 헬스 앱에서 벗어나, 식사가 주는 **즐거움(만족도), 식비 지출, 먹는 패턴, 그리고 감정적 포만감**을 종합적으로 분석하여 위트 있는 사용자 맞춤형 인사이트를 도출하는 것을 목표로 합니다.

---

## 2. 핵심 기능

### 2.1. 식사 기록 (Meal Logging)
- **식사 타임라인**: 
  - **식사 구분**: 아침, 점심, 저녁, 간식, 야식 중 선택.
  - **일시**: 식사한 날짜와 시간 기록 (클라이언트 사이드 하이드레이션 오류를 예방하기 위해 `suppressHydrationWarning`을 필수로 적용하여 표시).
- **식사 내용 및 메타데이터**:
  - **메뉴명**: 먹은 음식 이름 (최대 50자 제한, `maxLength` 방어적 UI 적용).
  - **식비**: 지출한 금액 입력 (식비 통계용).
  - **포만감 & 만족도**: 5단계 스케일(아이콘 기반) 제공.
  - **메모 및 감정 태그**: 식사 당시 분위기, 같이 먹은 사람, 기분 등을 한 줄 메모(최대 100자) 및 해시태그 형식으로 기록.
- **멀티미디어(음식 사진) 첨부**:
  - 게시글당 **최대 3장**의 음식 사진 첨부 가능.
  - **이미지 최적화**: 업로드 전 클라이언트 단에서 `Canvas` API를 활용하여 **최대 너비 800px의 WebP** 형식으로 자동 리사이징 후 업로드하여 스토리지 및 네트워크 대역폭 절약.
  - **LCP 및 안정성 보장**: Next.js `Image` 컴포넌트를 사용하고, 부모 요소에 `relative`를 필수로 지정하며 적절한 `sizes` 프롭을 할당하여 레이아웃 흔들림(CLS) 방지.
  - **SafeImage 적용**: 이미지 로드 실패 시, 깨진 엑스박스 대신 맛있는 도넛 모양의 **"음식 사진을 불러올 수 없습니다"** 전용 폴백 일러스트 표시.

### 2.2. 다차원 식사 통계 (Piggy Analytics)
- **칼로리 및 영양 분석 (선택 사항/수동 입력)**:
  - 사용자가 대략적으로 기입한 탄수화물, 단백질, 지방 비율 시각화.
- **식비 분석**:
  - 최근 7일/30일 동안 지출한 누적 식비.
  - 끼니별(아침/점심/저녁/야식) 평균 지출 및 최고 지출 메뉴.
- **식생활 습관 진단 (Insight Board)**:
  - **야식 주의보**: 밤 10시 이후 식사(야식) 비율 분석.
  - **식사 규칙성**: 하루 세 끼의 식사 시간대 분포도 시각화.
  - **최애 메뉴**: 가장 자주 먹은 음식 Top 5 및 해당 메뉴의 만족도 결합 분석.
  - **기분과 식사**: 특정 감정 태그(ex: #스트레스, #행복, #혼밥)가 식비와 포만감에 미치는 상관관계 분석.
- **주간 '돼지 지수' 리포트**:
  - 한 주간의 식사 기록을 종합하여 "미식가 돼지", "가성비 돼지", "야식 매니아 돼지", "건강 지킴이 돼지" 등 재미있는 위트 캐릭터 유형으로 분석하여 공유 카드로 제공.

### 2.3. 낙관적 업데이트 기반의 인라인 제어 (Optimistic UI)
- **즉각적인 반영**: 식사 추가, 수정 및 삭제 시 `useOptimistic`과 `startTransition` 콜백 내부에서 상태를 즉각 반영하여 네이티브 앱 수준의 빠른 반응성 제공.
- **인라인 간편 편집**: 타임라인 뷰에서 개별 식사 기록 카드를 클릭하면, 페이지 이동 없이 간편하게 금액이나 만족도를 즉시 토글/수정할 수 있는 미니 폼으로 인라인 전환.

## 3. 기술적 구현 및 아키텍처

### 3.1. 데이터베이스 스키마 설계 (Supabase + Drizzle ORM)
- **`meals` 테이블**:
  - `id`: uuid (Primary Key)
  - `user_id`: uuid (Supabase Auth reference)
  - `menu_name`: varchar(50) (Not Null)
  - `meal_type`: varchar(10) (Not Null - `breakfast`, `lunch`, `dinner`, `snack`, `night_snack`)
  - `price`: integer (Default 0, Nullable)
  - `satiety`: integer (1-5, 포만감)
  - `satisfaction`: integer (1-5, 만족도)
  - `memo`: varchar(100) (Nullable)
  - `tags`: text[] (Nullable)
  - `eaten_at`: timestamp with time zone (Not Null)
  - `created_at`: timestamp with time zone (Default now)
  - `updated_at`: timestamp with time zone (Default now)
- **`meal_images` 테이블**:
  - `id`: uuid (Primary Key)
  - `meal_id`: uuid (Foreign Key references `meals.id` ON DELETE CASCADE)
  - `image_path`: text (Supabase Storage path)
  - `created_at`: timestamp with time zone (Default now)

### 3.2. Supabase Storage 및 보안 정책 (RLS)
- **버킷**: `meal-images` 퍼블릭 버킷 생성.
- **보안**: RLS(Row Level Security)를 활성화하여 본인이 작성한 식사 기록 및 첨부된 이미지만 생성/수정/삭제 가능하도록 제한.
- **이미지 업로드 가이드라인**:
  - Client-side Canvas Resizer를 통해 저용량 WebP 파일로 전처리.
  - 업로드 전 DB 트랜잭션과 Storage 업로드 에러 시 롤백 정책 수립.

### 3.3. Next.js 15 서버 액션 (Server Actions) & TDD 검증
- 모든 식사 CRUD 관련 처리는 Server Actions를 통해 제어하며, 다음과 같은 구체적 예외 케이스 및 전수 테스트를 구현합니다.
- **성공 시나리오**:
  - 정상적인 데이터 입력 및 사진 업로드 성공.
- **최소 3개 이상의 실패/예외 시나리오**:
  1. **유효성 검사 실패**: 필수 필드 누락(메뉴명 공백) 또는 글자 수 제한 초과(`maxLength` 우회 입력 시도).
  2. **권한 부족**: 타인의 식사 기록을 수정/삭제하려고 시도하거나 비로그인 사용자의 요청 차단.
  3. **인프라(스토리지) 에러**: 이미지 파일 업로드 실패 시, 기존 DB 트랜잭션을 롤백하고 사용자에게 명확한 실패 에러 메시지(Alert) 전달.

## 4. UI/UX 가이드라인 (Design Aesthetics)

- **디자인 컨셉**: 슬림하고 컴팩트한 다크 모드/라이트 모드 하이브리드. 식욕을 자극하는 따뜻한 오렌지-피치 톤과 포근한 크림색 계열의 서브 컬러를 조합하여 아늑한 다이어리 느낌 전달.
- **식사 등록 모달 / 인라인 편집**:
  - 슬라이드 오버 또는 다이얼로그 모달 형식으로 깔끔하게 연출.
  - 슬라이더 또는 스마일리 이모티콘을 활용한 포만감/만족도 슬라이딩 선택기 구현.
- **통계 대시보드 시각화**:
  - 복잡한 라이브러리 사용을 최소화하고 HTML5 Canvas 또는 Tailwind CSS 트랜지션을 활용한 미니멀 커스텀 차트(막대, 원형 등)를 구현하여 성능 극대화 및 즉각적인 인터랙션 보장.
  - "이 주의 먹방 보고서" 형태의 카드 레이아웃.

## 5. 로드맵 (Roadmap)

- **1단계 (기반 구축 - 식사 기록)**:
  - `meals`, `meal_images` DB 마이그레이션 수립.
  - 캘린더/타임라인 UI 기반의 식사 CRUD API 및 컴포넌트 구현.
  - Canvas 크기 조정기가 탑재된 사진 업로드 시스템 구현.
- **2단계 (통계 및 대시보드 - Piggy Analytics)**:
  - 기간별 누적 식비, 최애 메뉴, 야식 비율 집계 쿼리 및 통계 화면 구현.
  - 위젯 형태의 커스텀 인터랙티브 차트 구현.
  - 요약 리포트 카드 디자인 및 공유(클립보드 복사) 기능 제공.
- **3단계 (고도화 및 UX 폴리싱)**:
  - 식단 분석 알고리즘 고도화 (인사이트 메시지 세분화).
  - 전체 기능에 대한 TDD 테스트 커버리지 확충 및 통합 빌드 검증 (`npm run verify`).
