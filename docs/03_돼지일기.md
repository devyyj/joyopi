# 돼지 일기 (Piggy Diary) - 상세 기능 명세 (v1.0 - Release)

## 1. 개요
'돼지 일기'는 사용자가 매일 먹은 음식과 식사를 기록하고, 이를 기반으로 직관적이고 재미있는 식생활 통계를 제공하는 조요피 연구소의 웰니스/라이프 스타일 서비스입니다. 

입력 피로감을 극대화하는 탄단지 칼로리 계산이나 딱딱한 식단 가이드에서 완전히 벗어났으며, 한 단계 더 나아가 **사용자의 데이터 기입 피로도를 극적으로 줄이기 위해 음식 가격(식비) 및 감정적 포만감(satiety) 지표를 시스템 전반에서 완전히 걷어냈습니다.** 오직 **식사가 주는 즐거움(만족도), 먹방 패턴, 최애 메뉴와 안 먹은 지 오래된 맛**을 종합적으로 분석하여 위트 있고 미니멀한 사용자 맞춤형 웰니스 인사이트를 도출합니다.

## 2. 핵심 기능

### 2.1. 극도의 미니멀 식사 기록 (Meal Logging)
- **식사 타임라인**: 
  - **식사 구분**: 아침, 점심, 저녁, 간식, 야식 중 선택.
  - **일시**: 식사한 날짜와 시간 기록 (클라이언트 사이드 하이드레이션 오류를 예방하기 위해 `suppressHydrationWarning`을 필수로 적용하여 표시).
- **식사 내용 및 메타데이터 (입력 최소화)**:
  - **메뉴명**: 먹은 음식 이름 (최대 50자 제한, `maxLength` 방어적 UI 적용).
  - **식비 및 포만감 필드 완전 배제**: 입력 스트레스를 없애기 위해 금액과 포만감 수치 기입란을 완전히 제거했습니다.
  - **만족도**: 5단계 스케일(아이콘 기반) 제공.
  - **메모 및 감정 태그**: 식사 당시 분위기, 같이 먹은 사람, 기분 등을 한 줄 메모(최대 100자) 및 클릭하기 편한 템플릿 태그 칩(#집밥, #혼밥, #야식, #스트레스 등) 형태로 쉽게 기록.
- **멀티미디어(음식 사진) 첨부 (선택 사항)**:
  - 사진을 꼭 올리지 않아도 텍스트와 만족도만으로 간편히 등록이 가능합니다.
  - **이미지 최적화**: 업로드 전 클라이언트 단에서 `Canvas` API를 활용하여 **최대 너비 800px의 WebP** 형식으로 자동 리사이징 후 업로드하여 스토리지 및 네트워크 대역폭 절약.
  - **LCP 및 안정성 보장**: Next.js `Image` 컴포넌트를 사용하고, 부모 요소에 `relative`를 필수로 지정하며 적절한 `sizes` 프롭을 할당하여 레이아웃 흔들림(CLS) 방지.
  - **SafeImage 적용**: 이미지 로드 실패 시 또는 이미지가 첨부되지 않은 기록의 경우, 귀여운 도넛 모양의 **폴백 기본 일러스트** 표시.

### 2.2. 고품격 식사 통계 (Piggy Analytics)
- **수평 끼니별 빈도 CSS 바 차트**:
  - 최근 7일/30일 동안의 끼니별(아침/점심/저녁/간식/야식) 먹방 비율을 가볍고 반응성이 뛰어난 동적 게이지로 시각화합니다.
- **이색 먹방 습관 진단**:
  - **야식 주의보**: 밤 10시부터 새벽 4시 사이의 식사(야식) 비율을 계산해 시각적 경고 멘트를 제공합니다.
  - **최애 메뉴 👑**: 2회 이상 가장 자주 중복으로 섭취한 최고의 단골 식단을 집계합니다.
  - **그리운 맛 🕰️**: 한동안 먹지 않은 지 가장 오래된 메뉴를 찾아내어 식단 다채로움을 모니터링합니다.
- **주간 '돼지 지수' 리포트**:
  - 한 주간의 식사 기록을 종합하여 "미식가 황제 돼지"(만족도 최상), "야식 파이터 돼지"(야식 비중 40% 이상), "소식 웰빙 돼지"(가벼운 식사 패턴), "가성비 요정 돼지" 등 위트 있는 캐릭터 유형으로 분석하여 멋진 요약 리포트 카드로 구성하고, 단 한 번의 터치로 **결과 공유하기 (클립보드 공유)** 기능을 지원합니다.

### 2.3. 세련된 '모던 플랫 링(도넛) 만족도 차트' 🍩
- **Recharts 기반의 링 차트**:
  - 어지럽고 번잡했던 radialGradient 방사형 광택이나 SVG blur 발광 필터(Neon Glow)를 완전히 제거하고, 극단적인 플랫 솔리드 세그먼트로 구성했습니다.
  - 조각 사이에는 `paddingAngle={2.5}`의 간격을 부여하고, 단정한 다크 톤 경계선(**`stroke="#1e1e24"`, `strokeWidth={1.5}`**)으로 세밀하게 테두리를 마감하여 여백의 미를 살렸습니다.
- **미니멀리즘 인터랙티브 페이드 효과**:
  - 호버 시 조각이 바깥으로 통 튀어나오던 어수선한 물리 애니메이션을 완전히 제거했습니다. 마우스 호버 시 해당 세그먼트가 선명해지며(`opacity-100 filter drop-shadow`), 다른 세그먼트는 부드럽게 반투명 처리(`opacity-40`)되는 은은한 투명도 변화만 주어 시각적 안정감을 극대화했습니다.
- **글래스모피즘 동적 피드백 보드 (`satisfaction-detail-feedback`)**:
  - 뚱뚱한 툴팁 대신 차트 바로 아래에 글래스모피즘 동적 안내 영역을 결합했습니다. 마우스를 링 세그먼트에 가져가면 해당 평점의 비율 데이터와 위트 넘치는 국문 정성 피드백이 실시간 전환됩니다. (호버하지 않을 때는 가장 우세한 만족도 평점이 기본 뷰로 예쁘게 안착하여 레이아웃 흔들림이 전혀 없습니다.)
- **고가독성 5색 골드 파스텔 배색 피팅 (눈부심 제로)**:
  - 5점의 노랑 색상이 밝아 가독성이 저하되던 점을 완전 보완하여, 눈이 최고로 편안하고 품격 있는 **파스텔 골드 옐로우(`#fbbf24`)**로 일괄 피팅했습니다. 링 차트와 하단의 스코어 칩 범례 뱃지가 동일 테마로 완벽히 어우러집니다.
    *   5점 👑: `#fbbf24` (고가독성 파스텔 골드)
    *   4점 🥰: `#fb923c` (따뜻한 피치 오렌지)
    *   3점 😋: `#c084fc` (세련된 퍼플)
    *   2점 🥱: `#818cf8` (차분한 인디고)
    *   1점 😭: `#f87171` (경고의 레드)

## 3. 기술적 구현 및 아키텍처

### 3.1. 데이터베이스 스키마 설계 (Supabase + Drizzle ORM)
- **`meals` 테이블**:
  - `id`: serial (Primary Key)
  - `userId`: uuid (profiles.id reference, ON DELETE CASCADE)
  - `menuName`: text (Not Null, 최대 50자 제한)
  - `mealType`: text (Not Null - `breakfast`, `lunch`, `dinner`, `snack`, `night_snack`)
  - `satisfaction`: integer (1-5, 만족도)
  - `memo`: text (최대 100자 제한, Nullable)
  - `tags`: text[] (Nullable)
  - `eatenAt`: timestamp with time zone (Not Null)
  - `createdAt`: timestamp (Default now)
  - `updatedAt`: timestamp (Default now)
- **`meal_images` 테이블**:
  - `id`: serial (Primary Key)
  - `mealId`: integer (meals.id reference, ON DELETE CASCADE)
  - `url`: text (Supabase Storage Public URL)
  - `createdAt`: timestamp (Default now)

### 3.2. Next.js 15 서버 액션 (Server Actions) & TDD 검증
- 모든 식사 CRUD 및 통계 집계는 Server Actions를 통해 고속 작동하며, TDD 원칙에 따라 철저한 유닛 테스트 케이스들을 통과해야 합니다.
- **검증된 3대 예외 시나리오**:
  1. **유효성 검사 실패**: 필수 필드 누락(메뉴명 공백) 또는 글자 수 제한 초과 우회 입력 시도 차단.
  2. **권한 부족**: 타인의 식사 기록을 임의로 수정/삭제하거나 비로그인 사용자의 기록 요청 철저히 방어.
  3. **스토리지(인프라) 에러**: 이미지 업로드 실패 시, 기존 DB 트랜잭션을 롤백하고 사용자에게 명확한 에러 얼럿 전달.

## 4. UI/UX 가이드라인 (Design Aesthetics)

- **디자인 컨셉**: 식욕을 돋우는 은은한 오렌지/피치 톤과 포근한 크림색 계열의 서브 컬러 조화. 다크 모드 기반의 부드러운 글래스모피즘 디자인 요소를 덧입혀 고급스럽고 위트 있는 웰니스 다이어리 연출.
- **Optimistic UI (낙관적 업데이트)**:
  - 식사 추가, 삭제 시 `useOptimistic` 및 `startTransition` 콜백을 적용하여, 로딩 딜레이 없이 UI 상태가 즉각 선반영되는 고속 반응성 실현.
  - 타임라인 카드 클릭 시 즉시 인라인으로 만족도 평점을 클릭 수정할 수 있는 **인라인 간편 편집 미니 폼** 활성화.
- **TDD 컴포넌트 검증**:
  - `piggy-analytics.test.tsx` 컴포넌트 테스트 스위트를 구비하여, Recharts 링 차트의 모킹 마운트와 5색 파스텔 컬러 렌더링, 하단 피드백 칩과의 동적 인터랙션이 JSDOM 환경에서도 100% 에러 없이 작동하는지 전수 점검.
