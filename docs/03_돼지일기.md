# 돼지 일기 (Piggy Diary) - 상세 기능 명세 (v1.1 - 현행화)

## 1. 개요
'돼지 일기'는 사용자가 매일 먹은 음식과 식사를 기록하고, 이를 기반으로 직관적이고 재미있는 식생활 통계를 제공하는 조요피 연구소의 웰니스/라이프 스타일 서비스입니다. 

입력 피로감을 극대화하는 탄단지 칼로리 계산이나 딱딱한 식단 가이드에서 완전히 벗어났으며, **사용자의 데이터 기입 피로도를 극적으로 줄이기 위해 음식 가격(식비) 및 감정적 포만감(satiety) 지표를 시스템 전반에서 완전히 걷어냈습니다.** 오직 **식사가 주는 즐거움(만족도), 먹방 패턴, 최애 메뉴와 안 먹은 지 오래된 맛**을 종합적으로 분석하여 위트 있고 미니멀한 사용자 맞춤형 웰니스 인사이트를 도출합니다.

## 2. 핵심 기능

### 2.1. 극도의 미니멀 식사 기록 (Meal Logging)
- **식사 타임라인**: 
  - **식사 구분**: 아침, 점심, 저녁, 간식, 야식 중 선택.
  - **일시**: 식사한 날짜와 시간 기록 (클라이언트 사이드 하이드레이션 오류를 예방하기 위해 `suppressHydrationWarning`을 필수로 적용하여 표시).
- **식사 내용 및 메타데이터 (입력 최소화)**:
  - **메뉴명**: 먹은 음식 이름 (최대 50자 제한, `maxLength` 방어적 UI 적용).
  - **식비 및 포만감 필드 완전 배제**: 입력 스트레스를 없애기 위해 금액과 포만감 수치 기입란을 완전히 제거했습니다.
  - **만족도**: 5단계 슬라이더(1~5점, 이모지 피드백) 제공. (1점 😭 ~ 5점 👑)
  - **감정 태그**: 식사 당시 분위기, 같이 먹은 사람, 기분 등을 클릭하기 편한 템플릿 태그 칩(`#혼밥`, `#스트레스`, `#행복`, `#가족`, `#야식`, `#배달음식`, `#치팅데이`, `#소소한행복`) 형태로 쉽게 선택. 직접 커스텀 태그 추가도 가능.
  - **한 줄 메모**: 식사 분위기나 에피소드를 자유롭게 기록 (최대 100자).
- **날짜 범위 필터 & 끼니 구분 필터**:
  - `DateRangeSelector` 컴포넌트를 통해 최근 7일 / 30일 / 전체 기간 또는 직접 날짜 지정 가능.
  - 끼니별(아침/점심/저녁/간식/야식/전체) 필터링 지원.
  - 커서 기반 페이지네이션(무한 스크롤)으로 타임라인 더보기 지원.
- **멀티미디어(음식 사진) 첨부 (선택 사항)**:
  - 사진을 꼭 올리지 않아도 텍스트와 만족도만으로 간편히 등록이 가능합니다.
  - **이미지 최적화**: 업로드 전 클라이언트 단에서 `Canvas` API를 활용하여 **최대 너비 800px의 WebP** 형식으로 자동 리사이징 후 업로드하여 스토리지 및 네트워크 대역폭 절약.
  - **최대 3장**: 1개 기록에 음식 사진 최대 3장 첨부 가능.
  - **LCP 및 안정성 보장**: Next.js `Image` 컴포넌트를 사용하고, 부모 요소에 `relative`를 필수로 지정하며 적절한 `sizes` 프롭을 할당하여 레이아웃 흔들림(CLS) 방지.
  - **SafeImage 적용**: 이미지 로드 실패 시 또는 이미지가 첨부되지 않은 기록의 경우, 귀여운 도넛 모양의 **폴백 기본 일러스트** 표시.

### 2.2. 고품격 식사 통계 (Piggy Analytics)
통계 대시보드는 `PiggyAnalytics` 컴포넌트로 구성되며, 3개의 카드로 이루어집니다.

#### 2.2.1. 최애 메뉴 & 그리운 맛 TOP 5 카드
- **최애 메뉴 TOP 5 👑**: 선택 기간 내 2회 이상 가장 자주 섭취한 메뉴를 횟수 내림차순으로 집계합니다. 순위별 금/은/동 컬러로 강조.
- **그리운 맛 TOP 5 🕰️**: 기록된 메뉴들 중 마지막으로 먹은 날이 가장 오래된 메뉴를 집계합니다. "N일 전" 또는 "오늘"로 표기.
- 두 목록이 모두 비어 있을 경우 빈 상태 UI 표시.

#### 2.2.2. 끼니별 먹방 빈도 차트 카드
- **Recharts `BarChart` (수평 가로 막대)**: 선택 기간 내 끼니별(아침/점심/저녁/간식/야식) 식사 횟수를 수평 막대 그래프로 시각화합니다.
  - 끼니별 파스텔 컬러 그라데이션(빨강 → 핑크 → 퍼플 → 인디고 → 블루) 적용.
  - 커스텀 툴팁으로 마우스 오버 시 끼니명과 횟수(회) 노출.
- **야식 주의보**: 차트 하단에 밤 10시(22:00)부터 새벽 4시(04:00) 사이의 식사 비율을 퍼센트로 표기. 40% 이상 시 붉은색 경고 색상으로 강조.

#### 2.2.3. 식사 만족도 분포 도넛 차트 카드
- **Recharts `PieChart` (도넛/링 형태)**:
  - 기록된 만족도(1~5점) 비율을 플랫 솔리드 세그먼트 도넛 차트로 시각화합니다.
  - 조각 사이에 `paddingAngle={5}`, `stroke="none"`으로 깔끔하게 마감.
  - 마우스 호버 시 해당 세그먼트가 선명해지고(`opacity-100`), 다른 세그먼트는 반투명(`opacity-40`) 처리.
- **도넛 차트 중앙 동적 텍스트**:
  - 툴팁 대신 차트 정중앙에 현재 호버 중인(또는 가장 우세한) 만족도 평점의 라벨·퍼센트·기록 횟수를 실시간으로 표시.
  - 호버하지 않을 때는 가장 많이 기록된 평점이 기본값으로 표시.
- **5색 파스텔 골드 배색**:
  - 5점 👑: `#fbbf24` (고가독성 파스텔 골드)
  - 4점 🥰: `#fb923c` (따뜻한 피치 오렌지)
  - 3점 😋: `#c084fc` (세련된 퍼플)
  - 2점 🥱: `#818cf8` (차분한 인디고)
  - 1점 😭: `#f87171` (경고의 레드)

## 3. 기술적 구현 및 아키텍처

### 3.1. 데이터베이스 스키마 설계 (Supabase + Drizzle ORM)
- **`meals` 테이블**:
  - `id`: serial (Primary Key)
  - `userId`: uuid (profiles.id reference, ON DELETE CASCADE)
  - `menuName`: text (Not Null, 최대 50자 제한)
  - `mealType`: text (Not Null - `breakfast`, `lunch`, `dinner`, `snack`, `night_snack`)
  - `satisfaction`: integer (1-5, 만족도)
  - `memo`: text (최대 100자 제한, Nullable)
  - `tags`: text[] (Nullable)
  - `eatenAt`: timestamp with time zone (Not Null)
  - `createdAt`: timestamp (Default now)
  - `updatedAt`: timestamp (Default now)
- **`meal_images` 테이블**:
  - `id`: serial (Primary Key)
  - `mealId`: integer (meals.id reference, ON DELETE CASCADE)
  - `url`: text (Supabase Storage Public URL, `meal-images` 버킷)
  - `createdAt`: timestamp (Default now)

### 3.2. Next.js 15 서버 액션 (Server Actions) & TDD 검증
- 모든 식사 CRUD 및 통계 집계는 Server Actions를 통해 고속 작동하며, TDD 원칙에 따라 철저한 유닛 테스트 케이스들을 통과해야 합니다.
- **검증된 3대 예외 시나리오**:
  1. **유효성 검사 실패**: 필수 필드 누락(메뉴명 공백) 또는 글자 수 제한 초과 우회 입력 시도 차단.
  2. **권한 부족**: 타인의 식사 기록을 임의로 수정/삭제하거나 비로그인 사용자의 기록 요청 철저히 방어.
  3. **스토리지(인프라) 에러**: 이미지 업로드 실패 시, 기존 DB 트랜잭션을 롤백하고 사용자에게 명확한 에러 얼럿 전달.

### 3.3. 페이지 아키텍처
- **`meals/page.tsx`**: 클라이언트 컴포넌트. Supabase 인증 상태 감지, 데이터 패칭(`getMeals` / `getMealStats`), 날짜 범위 및 끼니 필터 상태 관리.
- **`DateRangeSelector`**: 날짜 범위(7일/30일/전체/직접 지정) + 끼니 구분 필터 통합 컨트롤러.
- **`PiggyAnalytics`**: 3개 통계 카드(TOP5·바 차트·도넛 차트) 렌더링.
- **`MealTimeline`**: 커서 기반 무한 스크롤 타임라인. 인라인 만족도 편집 및 삭제 기능 포함.
- **`MealFormModal`**: 식사 등록/수정 모달 폼. 이미지 압축(Canvas WebP) 포함.
- **`Top5Stats`**: 최애 메뉴 TOP5 / 그리운 맛 TOP5 독립 컴포넌트 (테스트 커버리지 포함).

## 4. UI/UX 가이드라인 (Design Aesthetics)

- **디자인 컨셉**: 식욕을 돋우는 은은한 오렌지/피치 톤과 포근한 크림색 계열의 서브 컬러 조화. 다크 모드 기반의 부드러운 글래스모피즘 디자인 요소를 덧입혀 고급스럽고 위트 있는 웰니스 다이어리 연출.
- **Optimistic UI (낙관적 업데이트)**:
  - 식사 추가, 삭제 시 `useOptimistic` 및 `startTransition` 콜백을 적용하여, 로딩 딜레이 없이 UI 상태가 즉각 선반영되는 고속 반응성 실현.
  - 타임라인 카드 클릭 시 즉시 인라인으로 만족도 평점을 클릭 수정할 수 있는 **인라인 간편 편집 미니 폼** 활성화.
- **플로팅 액션 버튼(FAB)**:
  - 우측 하단에 항상 고정된 연필 모양 FAB 버튼으로 언제든 식사 기록 모달을 빠르게 호출 가능.
- **TDD 컴포넌트 검증**:
  - `piggy-analytics.test.tsx`, `top5-stats.test.tsx`, `date-range-selector.test.tsx` 테스트 스위트를 구비하여, Recharts 차트 모킹 마운트, TOP5 렌더링, 날짜 필터 동작이 JSDOM 환경에서도 에러 없이 작동하는지 전수 점검.
