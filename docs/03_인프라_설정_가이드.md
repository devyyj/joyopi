# 인프라 설정 가이드

이 문서는 프로젝트의 인프라 및 도구 설정에 관한 주요 변경 사항과 문제 해결 방법을 기록합니다.

## 1. Supabase MCP 인증 설정 문제 해결 (2026-05-18)

### 문제 상황
- Gemini CLI에서 Supabase MCP 도구 사용 시 인증 실패 발생.
- 전역(Global) 설정을 완료했음에도 불구하고 인증이 정상적으로 작동하지 않음.

### 원인 분석
1. **잘못된 패키지 명칭**: 기존 설정에서 `@supabase/mcp-server`를 사용하고 있었으나, 정식 패키지 명칭은 `@supabase/mcp-server-supabase`입니다.
2. **잘못된 환경 변수 이름**: PAT 인증 시 MCP 서버가 기대하는 환경 변수 이름은 `SUPABASE_ACCESS_TOKEN`이나, 기존에는 `SUPABASE_PAT` 또는 다른 이름을 사용하고 있었습니다.
3. **환경 변수 치환 오해**: 이전 가이드에서는 Gemini CLI가 환경 변수 치환을 지원하지 않는다고 기록되었으나, `settings.json`의 `env` 블록에서는 `$VARIABLE_NAME` 형식을 통한 환경 변수 참조가 정상적으로 지원됩니다.

### 해결 방법
- 전역 `settings.json`(`~/.gemini/settings.json`)을 다음과 같이 수정하여 인증 성공을 보장함:
    - 패키지 명칭: `@supabase/mcp-server-supabase@latest`
    - 환경 변수: `SUPABASE_ACCESS_TOKEN`에 실제 개인 액세스 토큰(PAT) 값을 직접 기입(하드코딩).
- 로컬 `.gemini/settings.json`을 비워 전역 설정이 우선 적용되도록 유지함.

### 적용된 설정 예시 (전역)
```json
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": ["-y", "@supabase/mcp-server-supabase@latest"],
      "env": {
        "SUPABASE_ACCESS_TOKEN": "sbp_your_actual_token_here"
      }
    }
  }
}
```

### 향후 가이드
- **인증 성공 우선**: 환경 변수 참조 방식에서 문제가 발생할 경우, 전역 `settings.json`에 토큰을 직접 기입하여 즉각적으로 문제를 해결할 수 있습니다.
- **보안 주의**: 전역 설정 파일은 로컬 프로젝트 Git에 포함되지 않으므로 코드 저장소를 통한 유출 위험은 적으나, 로컬 시스템 보안에 유의하십시오.
- **정식 명칭 확인**: MCP 서버 추가 시 반드시 공식 문서나 npm 저장소를 통해 정확한 패키지 명칭을 확인하십시오.
